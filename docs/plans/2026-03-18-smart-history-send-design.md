# 智能历史发送设计

## 问题描述
- Terminal 页面刷新后，之前的操作记录完全丢失
- 之前的修复为了避免 tmux attach 和乱码问题，只发送一个换行符
- 用户希望在刷新后能看到部分历史记录，且经常使用 tmux

## 解决方案

### 智能历史发送策略
根据输出内容自动检测是否在 tmux 会话中，决定发送多少历史：

**检测逻辑：**
- 检查缓冲区是否包含 "tmux" 或 "%0"（tmux 特征字符串）
- 如果检测到 tmux：只发送最近 50 行历史
- 如果未检测到 tmux：发送最近 500 行历史

### 实现位置
`backend/src/plugins/socket.io.ts` 的 `existingSession` 处理块

### 数据流程
```
页面刷新 → Terminal组件挂载 → 连接WebSocket
                                    ↓
后端收到connect-ssh → 查找现有session → 检测tmux特征
                                    ↓
发送对应数量的历史 → 注册新处理器 → 发送connected
```

## 优势
1. **自动适配**：根据环境自动调整历史量
2. **tmux 友好**：tmux 会话只发送少量历史，避免乱码
3. **用户体验好**：普通会话能看到足够的历史记录
4. **实现简单**：核心改动只在一个文件中
